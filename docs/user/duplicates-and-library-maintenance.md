# Duplicates and library maintenance

---

## Duplicate songs during scan

When **File → Scan Library** finds the same song in multiple locations, a **Duplicate song detected** or batch review dialog may appear.

Review side-by-side metadata and choose which copy to keep indexed.

---

## Duplicate folder structures {#duplicate-folders}

**File → Analyze duplicate folders…** finds clusters of folders with mirrored structure (common when copying Music trees).

For each cluster:

- Pick the folder to **keep**
- Unindex or trash duplicates

Resolve folder duplicates before chasing individual file duplicates.

---

## Set Export directory

The **Set Export directory** is intentionally **not scanned** so exported copies do not create duplicate library entries. See [Folder rules](settings/folder-rules.md).

---

## Re-scan after cleanup

Run **File → Scan Library** again after moving, deleting, or unindexing files so the database stays accurate.
